# Mission: Build a Tiny Local RAG Assistant (30–60 min)

Tags: #learning #mission #ai #devops #science
Links: [[Home]]

## Objective
Create a minimal local Retrieval-Augmented Generation (RAG) prototype that answers questions from your own notes/files, and document what worked.

## Exact Steps
1. **Create a working folder** (e.g., `~/tmp/rag-mini`) and add 3–5 short text files as your knowledge base.
2. **Set up environment**:
   - `python -m venv .venv && source .venv/bin/activate`
   - `pip install sentence-transformers faiss-cpu`
3. **Build embeddings + index**:
   - Write `build_index.py` to load files, chunk text (small chunks), create embeddings, and store a FAISS index + metadata.
4. **Query flow**:
   - Write `ask.py` that embeds a question, retrieves top-3 chunks, and prints:
     - matched chunks
     - a final synthesized answer (simple template-based summary is fine).
5. **Test with 3 questions** and record whether retrieval was relevant.
6. **Write a short reflection**: what was easiest, what broke, what to improve next.

## Expected Output Artifact
- A folder containing:
  - `build_index.py`
  - `ask.py`
  - `index.faiss` + metadata file
  - `README.md` with run instructions and 3 example Q&A results

## Stretch Challenge
Containerize it with a `Dockerfile` and run the full pipeline (`build_index.py` then `ask.py`) inside Docker for reproducibility.
