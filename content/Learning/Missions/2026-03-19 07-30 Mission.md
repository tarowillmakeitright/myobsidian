# Mission: Prompt Caching Benchmark (AI + DevOps)

Tags: #learning #mission #ai #devops #science
Links: [[Home]]
Date: 2026-03-19 07:30 (Asia/Tokyo)

## Objective
Measure how much latency and token cost you can save in one local CLI workflow by reusing a structured prompt template with cache-friendly static sections.

## Exact Steps (30–60 min)
1. **Pick one repeated task** (e.g., summarize logs, draft commit message, explain test failure).
2. **Create two prompt versions** in a note or text file:
   - **A (baseline):** one large free-form prompt each run.
   - **B (cache-friendly):** stable instruction block + small variable input block.
3. **Run each version 5 times** on similar inputs (10 runs total).
4. **Record for each run:** response time, input tokens, output tokens, and estimated cost (from your tool output/dashboard).
5. **Compute averages** for A vs B (time + token usage).
6. **Write 3 bullet conclusions**: what improved, what did not, and where to apply this pattern next.

## Expected Output Artifact
A markdown table with 10 runs and a short findings section, saved as:
`Learning/Missions/2026-03-19 Prompt Caching Results.md`

## Stretch Challenge
Automate the benchmark with a small script (bash or Python) that runs both prompt variants and outputs a CSV summary automatically.
