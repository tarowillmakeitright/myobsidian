# Daily Learning Mission (2026-02-27)

[[Home]]

## Mission Title
Build a Minimal AI Log Anomaly Detector with Python

## Objective
In 30–60 minutes, create a small script that scans a server log file, flags unusual lines using simple heuristics, and outputs a short anomaly report.

## Exact Steps
1. Create a working folder and a sample log file (`app.log`) with 30–50 lines (mostly normal, a few suspicious: repeated failures, spikes, odd IPs).
2. Create `detect_anomalies.py`.
3. Parse each log line and extract: timestamp, log level, message, and source IP (if present).
4. Implement 3 heuristic rules:
   - More than 5 `ERROR` lines in any 5-minute window
   - More than 3 failed login messages from same IP in 10 minutes
   - Any log message containing keywords: `timeout`, `unauthorized`, `panic`
5. Generate `anomaly_report.md` with:
   - total lines scanned
   - number of anomalies by rule
   - top 3 suspicious IPs/messages
6. Run the script and verify report readability.

## Expected Output Artifact
- `detect_anomalies.py`
- `anomaly_report.md` summarizing detected anomalies

## Stretch Challenge
Add a tiny “AI assistant” step: send each flagged message to a local LLM/API prompt template that classifies severity (`low/medium/high`) and append severity to the report.
