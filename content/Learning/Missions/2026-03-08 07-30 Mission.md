## Mission Title
Build a Local LLM Prompt Benchmark Harness (AI + DevOps)

## Objective
In 30–60 minutes, create a tiny, repeatable benchmark that runs 3 prompts against one local or API model and records latency + output quality notes in a markdown table.

## Exact Steps
1. Create a folder for today’s benchmark (e.g., `~/tmp/llm-bench-2026-03-08`).
2. Prepare 3 prompts:
   - one summarization task,
   - one coding/debug task,
   - one reasoning/explanation task.
3. Run each prompt once (or twice if time allows) against your chosen model.
4. For each run, record:
   - prompt name,
   - start/end time (or elapsed seconds),
   - brief quality score (1–5),
   - one-line note on output usefulness.
5. Put results into a markdown table.
6. Write 3 bullet takeaways: what was fastest, what was best quality, and one change to test tomorrow.

## Expected Output Artifact
A markdown note named `benchmark-results-2026-03-08.md` containing:
- the 3 prompts,
- a results table (latency + quality),
- 3 short takeaways.

## Stretch Challenge
Automate one part: write a tiny script (bash/python) that timestamps each run automatically and appends a row to your results table.
