# Mission: Container Health Check Lab (DevOps)

[[Home]]

## Objective
Build a tiny Dockerized app and add a working health check, then verify container health transitions from `starting` to `healthy`.

## Exact Steps (30–60 min)
1. Create a folder `healthcheck-lab` with:
   - `app.py` (simple HTTP server with `/health` endpoint returning `200 OK`)
   - `Dockerfile` (Python slim image, copy app, run it)
2. Add a Docker `HEALTHCHECK` in the Dockerfile (or in `docker-compose.yml`) that calls `/health` every 10s.
3. Build and run the container:
   - `docker build -t healthcheck-lab .`
   - `docker run -d --name health-lab -p 8080:8080 healthcheck-lab`
4. Confirm behavior:
   - `curl http://localhost:8080/health`
   - `docker ps` and `docker inspect --format='{{json .State.Health}}' health-lab`
5. Briefly simulate failure (e.g., modify `/health` to return 500), rebuild, rerun, and observe `unhealthy`.

## Expected Output Artifact
A short note in this file with:
- one screenshot or pasted terminal output showing `healthy`
- one output showing `unhealthy`
- 3 bullet points: what worked, what failed, what you learned

## Stretch Challenge
Add a `docker-compose.yml` with `restart: unless-stopped` and tune healthcheck settings (`interval`, `timeout`, `retries`) to reduce false positives.
