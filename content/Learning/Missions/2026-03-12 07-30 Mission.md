## Mission Title
Prompt Regression Harness for LLM Outputs

## Objective
Build a tiny, repeatable test harness that checks whether prompt or model changes degrade output quality on 5 fixed test cases.

## Exact Steps (30–60 min)
1. Create a folder `llm-regression-mini/` and add a file `cases.json` with 5 prompts and expected quality criteria (e.g., must include 3 bullet points, no hallucinated URLs, <=120 words).
2. Write a simple script (Python or Node) that:
   - loops through the 5 prompts,
   - calls your chosen LLM/API,
   - stores outputs in `results/<timestamp>.json`.
3. Add an evaluator function using rule checks (length, required keywords, forbidden patterns) and compute a per-case pass/fail.
4. Generate a summary report `report.md` with:
   - total pass rate,
   - failed cases,
   - one-line suspected cause per failure.
5. Re-run once with a small prompt tweak (or different model setting) and compare pass rates.

## Expected Output Artifact
A mini repo/folder containing:
- `cases.json`
- runner script
- `results/*.json`
- `report.md` with before/after comparison

## Stretch Challenge
Add a CI step (GitHub Actions or local pre-commit script) that fails when pass rate drops below 80%.
