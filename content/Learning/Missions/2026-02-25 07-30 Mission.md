# Mission: Build a Local RAG Mini-Pipeline (30–60 min)

Tags: #learning #mission #ai #devops #science
Links: [[Home]]

## Objective
Create a tiny Retrieval-Augmented Generation (RAG) workflow that ingests one short technical note, embeds it, and answers 3 questions grounded in that note.

## Exact Steps
1. **Create input note (5 min):**
   - Make `note.txt` with ~200–400 words on one topic (e.g., Kubernetes probes, transformer attention, or Artemis mission timeline).
2. **Set up quick environment (10 min):**
   - Create a Python venv and install: `sentence-transformers`, `faiss-cpu`, `numpy`.
3. **Ingest + index (10–15 min):**
   - Split `note.txt` into chunks (~120–180 words).
   - Generate embeddings for each chunk.
   - Store embeddings in a FAISS index.
4. **Query + retrieve (10–15 min):**
   - Run 3 questions against your index.
   - Retrieve top-2 chunks for each question.
5. **Grounded answer check (5–10 min):**
   - For each question, produce an answer using only retrieved chunks.
   - Add a confidence score (High/Med/Low) and one citation line per answer.

## Expected Output Artifact
A single markdown report: `rag-mission-report.md` containing:
- The 3 questions
- Retrieved chunks (or chunk IDs)
- Final grounded answers
- Confidence + citations
- 3 bullet “what I learned” notes

## Stretch Challenge
Containerize it with a minimal `Dockerfile` and run the same pipeline in a container using one command.
