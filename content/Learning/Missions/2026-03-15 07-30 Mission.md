# Mission: Build a Tiny RAG Q&A Bot with Local Docs

Tags: #learning #mission #ai #devops #science
Links: [[Home]]

## Objective
In 30–60 minutes, build a minimal Retrieval-Augmented Generation (RAG) pipeline that answers questions from one local markdown file.

## Exact Steps
1. Create a working folder and add one markdown doc (your source knowledge).
2. Chunk the text into small passages (e.g., ~300–500 characters each).
3. Generate embeddings for each chunk (any embedding model/API).
4. Store vectors in a simple local index (FAISS/Chroma or similar).
5. Write a script that:
   - Accepts a user question
   - Retrieves top 3 relevant chunks
   - Sends question + retrieved context to an LLM
   - Prints answer + cited chunk snippets
6. Test with 3 questions (easy, medium, tricky) and note where retrieval fails.

## Expected Output Artifact
- One runnable script (`rag_demo.py` or `rag_demo.js`)
- One short `README.md` with setup + run commands
- A sample output log showing 3 Q&A runs with retrieved context

## Stretch Challenge
Add a “source confidence” line (e.g., average similarity score) and only answer when confidence is above a threshold; otherwise return “I’m not confident yet.”
