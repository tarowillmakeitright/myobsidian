# Mission: Build a Tiny RAG Evaluator (30–60 min)

Tags: #learning #mission #ai #devops #science
Links: [[Home]]

## Objective
Create a minimal local script that checks whether retrieval quality improves answer quality for a small Q&A set.

## Exact Steps
1. Create a folder `rag-mini-eval` and a Python file `eval.py`.
2. Add 5 short documents (plain text) about one topic (e.g., Kubernetes basics or Mars missions).
3. Write 5 matching questions with expected key points.
4. In `eval.py`, implement two modes:
   - **No retrieval**: answer from model directly.
   - **With retrieval**: select top-2 relevant docs by keyword overlap and prepend them as context.
5. Score each answer 0–2:
   - 0 = wrong,
   - 1 = partially correct,
   - 2 = correct key points.
6. Run both modes on all 5 questions and calculate average score.
7. Save results to `results.md` with a tiny table and 3 bullet insights.

## Expected Output Artifact
- `results.md` containing:
  - average score (no retrieval vs with retrieval),
  - per-question scores,
  - short conclusion on whether retrieval helped.

## Stretch Challenge
Replace keyword overlap with cosine similarity using sentence embeddings, then compare scores again.
