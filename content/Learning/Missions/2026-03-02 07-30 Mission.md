# Daily Learning Mission (2026-03-02 07:30)
[[Home]]

## Mission Title
**Log Guardian Lite: Detect unusual server log spikes with Python**

## Objective
Build a tiny script that scans an Nginx/Apache access log and flags unusual spikes in 5-minute request volume. Finish with one chart and one short interpretation.

## Exact Steps (30–60 min)
1. Create a working folder: `mkdir -p ~/tmp/log-mission && cd ~/tmp/log-mission`.
2. Get a sample access log (or copy one you already have) into `access.log`.
3. Create `analyze.py` that:
   - Parses timestamps from each log line.
   - Buckets requests into 5-minute windows.
   - Calculates mean and standard deviation of window counts.
   - Flags windows where `count > mean + 2*std`.
4. Add chart output using matplotlib:
   - Line chart of request count per 5-minute window.
   - Highlight flagged windows in red.
   - Save as `traffic_spikes.png`.
5. Run: `python3 analyze.py`.
6. Write 3 bullet insights in `notes.md`:
   - Total requests processed.
   - Number of anomalous windows.
   - Most likely cause hypothesis (bot burst, deploy, scraping, etc.).

## Expected Output Artifact
- `analyze.py`
- `traffic_spikes.png`
- `notes.md` (3 concise findings)

## Stretch Challenge
Convert the script into a CLI (`python3 analyze.py --file access.log --window 5 --sigma 2`) and output anomalies as JSON for future alerting integration.
