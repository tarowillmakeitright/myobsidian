# Mission: Build a Tiny RAG Evaluator (AI + DevOps)

Tags: #learning #mission #ai #devops #science
Links: [[Home]]

## Objective
In 30–60 minutes, create a minimal script that compares two retrieval settings for a small RAG pipeline and reports which one gives better top-3 relevance.

## Exact Steps
1. Create a folder: `rag-mini-eval/`.
2. Add 8–12 short documents (plain text) on one topic you like (e.g., Mars missions, Kubernetes basics).
3. Write 5 test questions with expected keywords for correct answers.
4. Implement a simple retriever (Python):
   - Setting A: basic keyword match (e.g., TF-IDF/BM25 library).
   - Setting B: same retriever but with query expansion (add 2–3 synonyms manually).
5. For each question, retrieve top-3 docs for A and B.
6. Score each setting: count how many top-3 results contain expected keywords.
7. Print a small comparison table and best setting.

## Expected Output Artifact
A single `results.md` containing:
- Questions
- Top-3 docs from A vs B
- Per-question score
- Final winner + 2–3 bullet insights

## Stretch Challenge
Add a tiny CI step (GitHub Actions or local pre-commit script) that reruns the evaluator and fails if average score drops below your baseline.
