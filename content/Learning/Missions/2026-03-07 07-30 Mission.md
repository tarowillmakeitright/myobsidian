## Mission Title
Mini Prompt Evaluation Harness (Local)

## Objective
In 30–60 minutes, build a tiny, repeatable evaluation workflow to compare 2 prompt variants for one real task you do often.

## Exact Steps
1. Pick one task (e.g., “summarize a technical article into 5 bullet points”).
2. Create a file `eval_cases.txt` with 5 short test inputs.
3. Write Prompt A (current style) and Prompt B (improved style with clearer constraints).
4. Run both prompts on all 5 inputs (manual or script) and save outputs as `results_A.md` and `results_B.md`.
5. Score each output (1–5) on: correctness, clarity, and actionability.
6. Write a short conclusion: where A wins, where B wins, and the final “default prompt.”

## Expected Output Artifact
A single note or repo folder containing:
- `eval_cases.txt`
- `results_A.md`
- `results_B.md`
- `prompt_eval_summary.md` (table of scores + final chosen prompt)

## Stretch Challenge
Automate scoring structure with a simple script (Python or Bash) that generates a template score table for each run, so you can reuse this evaluation process weekly.
