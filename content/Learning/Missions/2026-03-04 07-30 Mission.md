# Mission: Build a Tiny RAG Assistant with Local Embeddings

Tags: #learning #mission #ai #devops #science  
Links: [[Home]]

## Objective
In 30–60 minutes, build a minimal Retrieval-Augmented Generation (RAG) prototype that answers questions from one local markdown file.

## Exact Steps
1. Create a project folder and add one source file (`notes.md`) with ~15–30 lines of technical notes.
2. Set up Python env and install: `sentence-transformers`, `faiss-cpu`, `numpy`.
3. Split `notes.md` into chunks (about 300–500 chars each).
4. Generate embeddings for each chunk using `all-MiniLM-L6-v2`.
5. Store vectors in FAISS and keep chunk text in a list/map.
6. Write a script that:
   - takes a user question,
   - embeds it,
   - retrieves top-3 closest chunks,
   - prints them as context + a short synthesized answer.
7. Test with 3 questions and tune chunk size/top-k once.

## Expected Output Artifact
One runnable script (`mini_rag.py`) plus a short `README.md` with:
- how to run,
- sample questions,
- one screenshot or pasted terminal output showing retrieval + answer.

## Stretch Challenge
Containerize it with Docker and expose a tiny CLI command (`python mini_rag.py "your question"`) inside the container.
