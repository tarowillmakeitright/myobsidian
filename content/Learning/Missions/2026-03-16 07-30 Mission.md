## Mission Title
**Containerize and Deploy a Tiny AI Inference API (Local-First)**

## Objective
In 30–60 minutes, build a minimal FastAPI service that serves one AI-related endpoint, run it in Docker, and verify it with a curl test.

## Exact Steps
1. Create a new folder (e.g., `tiny-ai-api`) with:
   - `app.py` (FastAPI app)
   - `requirements.txt` (`fastapi`, `uvicorn`)
   - `Dockerfile`
2. Implement one endpoint in `app.py`:
   - `GET /health` → returns `{ "status": "ok" }`
   - `POST /score` → accepts `{ "text": "..." }` and returns a simple heuristic score (e.g., length-based) and label (`short`, `medium`, `long`).
3. Build and run locally:
   - `docker build -t tiny-ai-api .`
   - `docker run --rm -p 8000:8000 tiny-ai-api`
4. Validate with curl:
   - `curl http://localhost:8000/health`
   - `curl -X POST http://localhost:8000/score -H "Content-Type: application/json" -d '{"text":"hello world from tokyo"}'`
5. Add one short README section: purpose, run commands, sample request/response.

## Expected Output Artifact
- A working repo/folder containing `app.py`, `requirements.txt`, `Dockerfile`, and README.
- Proof of success: terminal output from both curl calls (health + score).

Add `docker-compose.yml` and a basic GitHub Actions workflow that builds the Docker image on every push.
