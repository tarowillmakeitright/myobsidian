# Mission: Build a Tiny RAG Evaluator for Hallucination Checks

Tags: #learning #mission #ai #devops #science
Links: [[Home]]

## Objective
Create a minimal local script that compares a model’s answer **with vs without retrieval context** and reports whether retrieval reduced hallucination risk.

## Exact Steps (30–60 min)
1. Create a folder `mini-rag-eval/` and add a `facts.txt` file with 8–12 short factual lines (pick one topic, e.g., Mars missions).
2. Write a small script (`evaluate.py` or `evaluate.js`) that:
   - accepts a question,
   - generates Answer A (no context),
   - generates Answer B (with top 3 retrieved lines from `facts.txt` via simple keyword matching),
   - scores each answer with a simple rubric: factual alignment (0–2), unsupported claims count.
3. Test on 5 questions (at least 2 tricky/ambiguous).
4. Log results in `results.md` with a tiny table: question, A score, B score, winner, note.
5. Add one paragraph: “When retrieval helped / failed and why.”

## Expected Output Artifact
A small repo/folder containing:
- `facts.txt`
- evaluator script
- `results.md` with 5 evaluated questions and conclusions

## Stretch Challenge
Add a confidence gate: if retrieval coverage is low, make the script return “insufficient evidence” instead of forcing an answer.
