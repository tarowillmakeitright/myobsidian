## Mission Title
Build a Tiny RAG Assistant with Local Docs (30–60 min)

## Objective
Create a minimal retrieval-augmented generation (RAG) workflow that answers questions from a small local markdown knowledge base.

## Exact Steps
1. Create a folder `rag-mini/` with two files: `notes1.md` and `notes2.md` (5–10 short facts each on one topic you care about).
2. Set up a quick Python script (`app.py`) that:
   - loads both markdown files,
   - splits text into chunks,
   - creates embeddings,
   - stores them in a lightweight vector store (e.g., FAISS/Chroma).
3. Add a query function: user asks one question, script retrieves top 3 relevant chunks.
4. Pass retrieved context + question to an LLM and generate an answer.
5. Print output in this format:
   - Question
   - Retrieved chunks (short)
   - Final answer
6. Run 3 test questions and verify each answer references relevant retrieved context.

## Expected Output Artifact
A GitHub gist or local project folder containing:
- `app.py`
- `requirements.txt`
- `notes1.md`, `notes2.md`
- one screenshot or terminal log showing 3 successful Q&A runs

## Stretch Challenge
Add a simple confidence guard: if top retrieval score is below a threshold, return “I don’t have enough context in the local notes.” instead of hallucinating.
