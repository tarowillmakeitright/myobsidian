## Mission Title
AI Ops Quickstart: Build a Local Log-Anomaly Detector

## Objective
In 30–60 minutes, create a tiny Python workflow that flags unusual log lines using sentence embeddings + cosine distance, then outputs the top anomalies.

## Exact Steps
1. Create a folder `log-anomaly-mission` and a sample file `logs.txt` with 40–60 normal app log lines plus 5 intentionally weird lines.
2. Set up environment:
   - `python -m venv .venv && source .venv/bin/activate`
   - `pip install sentence-transformers scikit-learn pandas`
3. Write `detect.py`:
   - Load lines from `logs.txt`
   - Embed with `all-MiniLM-L6-v2`
   - Compute centroid of embeddings
   - Score each line by cosine distance from centroid
   - Print top 10 most anomalous lines with scores
4. Run `python detect.py` and verify weird lines appear near the top.
5. Save outputs to:
   - `results/top_anomalies.txt`
   - `results/summary.md` (what worked, what failed, quick improvements)

## Expected Output Artifact
A mini repo/folder containing:
- `detect.py`
- `logs.txt`
- `results/top_anomalies.txt`
- `results/summary.md`

## Stretch Challenge
Containerize it with a `Dockerfile` and run:
`docker run --rm -v $(pwd):/app log-anomaly:latest`
so anomaly detection works reproducibly on any machine.
