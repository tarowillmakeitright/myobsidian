## Mission Title
Build and Run a Tiny Local RAG Pipeline (AI + DevOps)

## Objective
In 30–60 minutes, create a minimal Retrieval-Augmented Generation workflow that indexes 3 short docs and answers one question with cited context.

## Exact Steps
1. Create a folder `mini-rag` with three text files (`doc1.txt`, `doc2.txt`, `doc3.txt`) on topics you care about.
2. Write a small Python script (`rag_demo.py`) that:
   - loads the three docs,
   - splits text into chunks,
   - builds embeddings (use a lightweight local model/library),
   - stores vectors in a simple local DB (e.g., FAISS/Chroma),
   - retrieves top-2 chunks for a query,
   - prints: query, retrieved chunks, and final answer.
3. Add one `requirements.txt` and run it in a virtual environment.
4. Test with 2 queries; adjust chunk size or top-k once.
5. Save terminal output to `run-log.txt`.

## Expected Output Artifact
A working `mini-rag/` folder containing:
- `rag_demo.py`
- `requirements.txt`
- 3 source docs
- `run-log.txt` showing at least one successful query with retrieved context.

## Stretch Challenge
Containerize it with a minimal `Dockerfile` and run it once via `docker run`.
