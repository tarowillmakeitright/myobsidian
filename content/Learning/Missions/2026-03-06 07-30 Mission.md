## Mission Title
**Build a tiny RAG pipeline with local embeddings + evaluation**

## Objective (30–60 min)
Create a minimal Retrieval-Augmented Generation workflow that answers questions from a small custom note set, then quickly evaluate answer quality.

## Exact Steps
1. Create a small dataset (5–10 short notes) in a single `.md` or `.txt` file on one topic you care about.
2. Generate embeddings for each note chunk (use any familiar tool/library).
3. Store vectors in a simple local index (FAISS/Chroma or equivalent).
4. Run 5 test questions against retrieval + LLM answer generation.
5. Score each answer from 1–5 on: factuality, relevance, and completeness.
6. Note one failure pattern and one concrete improvement.

## Expected Output Artifact
A short write-up (`rag-mini-report.md`) containing:
- stack/tools used
- 5 Q&A test cases
- score table (or bullet scores)
- biggest failure + next fix

## Stretch Challenge
Add a reranking step (or metadata filtering) and compare before/after scores on the same 5 questions.
