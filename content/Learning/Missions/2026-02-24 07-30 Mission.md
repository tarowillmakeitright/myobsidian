# Daily Learning Mission — 2026-02-24
[[Home]]

## Mission Title
Build a Tiny RAG Evaluator (AI + DevOps-lite)

## Objective
In 30–60 minutes, create a minimal script that compares retrieval quality across two embedding models on a small local document set.

## Exact Steps
1. Create a folder `rag-mini-eval/` with 5–10 short markdown/text docs (topics you care about).
2. Prepare 5 test questions with expected “relevant doc(s)” noted in a JSON/YAML file.
3. Implement a simple pipeline (Python or Node):
   - chunk docs (fixed-size chunks),
   - embed chunks with **Model A**,
   - retrieve top-3 for each question,
   - repeat with **Model B**.
4. Compute a basic metric: Recall@3 (did expected doc appear in top-3?).
5. Output a comparison table and 3 short observations.

## Expected Output Artifact
A repo/folder containing:
- `eval.py` or `eval.js`
- `questions.json` (with expected docs)
- `results.md` (Model A vs B table + observations)

## Stretch Challenge
Add a lightweight CI check (GitHub Actions or local pre-commit script) that fails if Recall@3 drops below your baseline.
