# Daily Learning Mission — 2026-03-18
[[Home]]

## Mission Title
Build and Compare Two Lightweight RAG Retrieval Pipelines (BM25 vs Vector)

## Objective (30–60 min)
Create a tiny retrieval benchmark using ~20 local notes/docs and compare keyword search (BM25) against vector search for relevance and speed.

## Exact Steps
1. Create a folder with ~20 short docs (markdown/text) from your notes or sample content.
2. Define 5 realistic queries you might actually ask those docs.
3. Run BM25 retrieval (top-3 results per query) and record outputs.
4. Run vector retrieval (top-3 results per query) using any local embedding tool/library.
5. For each query, score relevance quickly (0/1 per result) and note retrieval latency.
6. Summarize which method performed better by query type (exact keyword vs semantic intent).

## Expected Output Artifact
A single comparison note: `Learning/Missions/2026-03-18 RAG Retrieval Comparison.md` containing:
- query list
- BM25 top-3 + scores + latency
- Vector top-3 + scores + latency
- 5–8 bullet conclusions and one practical “default strategy” recommendation

## Stretch Challenge
Add a hybrid retriever (BM25 + vector rerank or weighted merge) and test whether it beats both baselines on your 5 queries.
